무엇이 다른가?

구분 터미널 에뮬레이터 터미널 멀티플렉서 (tmux)
역할 GUI 화면 표시 (Ghostty, iTerm2) 터미널 내 세션/공간 관리
지속성 창 닫으면 프로세스 종료 창 닫아도 세션 유지 (Detached)
분할 제한적 탭 기능 무한한 윈도우/페인 분할

tmux는 그 안의 유연한 공간입니다."

속도, 폰트 지원, 심플함의 완벽한 조화

3계층 구조 멘탈 모델

🏢 SESSION (건물 전체: 프로젝트 단위)
🏢 WINDOW (각 층: 탭 단위)
🏢 PANE (각 방: 화면 분할 단위)

관심사의 분리

  • Session: 독립된 업무 공간. 퇴근해도 사라지지 않음.
  • Window: 에이전트별(Claude, Codex, Gemini) 분리.
  • Pane: 작업, 로그, 탐색기를 한 눈에 모니터링.

AI 에이전트 생산성 변화

수치가 증명하는 변화

tmux 도입 후 병렬 작업이 가능해지면서 AI 토큰 사용량이 수 배 증가했습니다. 이는 더 많은 에이전트를 동시에 돌려 검증 시간을 단축했음을 의미합니다.

  • 단일 작업 모드: 1x Speed
  • 멀티 페인 모니터링: 2.5x Speed
  • 스크립트 자동화: 5x+ Speed

필수 명령어 치트시트

기본 Prefix: Ctrl + B (이후 아래 키 입력)

c : 새 윈도우 생성
% : 수직 분할 (좌우)
" : 수평 분할 (위아래)
d : 세션 디태치
z : 페인 확대/축소
w : 목록 보기
$ : 세션 이름 변경
, : 윈도우 이름 변경
n/p : 다음/이전 이동
q : 페인 번호 표시

나만의 터미널 커스터마이징

.tmux.conf 설정

# Prefix를 Ctrl+A로 변경
set -g prefix C-a
unbind C-b
bind C-a send-prefix

# 마우스 활성화
set -g mouse on

# vi 키바인딩 (hjkl 이동)
setw -g mode-keys vi

# 히스토리 확장
set -g history-limit 50000

엔트로피 감소: 자동화 스크립트

아침에 출근해 명령어 하나로 전체 개발 환경을 1초 만에 세팅합니다.

#!/bin/bash
tmux new-session -d -s work
tmux rename-window -t work:0 'Dev'
tmux send-keys -t work:Dev 'nvim .' Enter

tmux new-window -t work -n 'Agents'
tmux split-window -h -t work:Agents
tmux send-keys -t work:Agents.0 'claude' Enter
tmux send-keys -t work:Agents.1 'codex' Enter

tmux attach -t work

상세 분석 요약

1. 배경 및 중요성 +
AI 에뮬레이터 시대, 병렬 모니터링과 세션 유지의 필요성 대두. 19년 된 도구의 화려한 귀환.
2. 에뮬레이터 vs 멀티플렉서 +
Ghostty와 tmux의 차이점을 명확히 인지하고 각자의 장점을 극대화하는 조합 추천.
3. 3계층 구조 실습 +
세션(Session) -> 윈도우(Window) -> 페인(Pane)의 위계를 실제 업무에 대입하는 방법.
4. 고급 자동화 +
셸 스크립트를 이용한 복합 환경 구성 및 AI 에이전트 오케스트레이션.

한 줄 요약 (Feynman)
