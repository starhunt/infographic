🔄 전송 계층 (Transport Layer)

// stdio 로컬 프로세스 통신 { "transport": "stdio", "process": { "command": "node", "args": ["server.js"], "env": {"NODE_ENV": "production"} } } // HTTP/SSE 원격 통신 { "transport": "http+sse", "endpoint": "https://api.mcp.com", "auth": "oauth2", "timeout": 30000 }
  • stdio: 자식 프로세스 spawn, IPC 채널 활용
  • HTTP+SSE: Server-Sent Events로 실시간 스트리밍
  • WebSocket: 양방향 실시간 통신 (개발중)

📊 데이터 계층 (Data Layer)

// JSON-RPC 2.0 메시지 구조 { "jsonrpc": "2.0", "method": "initialize", "params": { "protocolVersion": "0.1.0", "capabilities": { "tools": true, "resources": true, "prompts": true, "sampling": true }, "clientInfo": { "name": "claude-desktop", "version": "1.0.0" } }, "id": "init-001" }
  • 버전 협상: Semantic Versioning 기반 호환성 체크
  • 기능 탐지: Capabilities 협상으로 동적 기능 활성화
  • 에러 처리: 표준화된 에러 코드 체계 (-32700 ~ -32000)

MCP Primitives 심화 분석

LLM 상호작용을 위한 6가지 핵심 빌딩 블록의 기술적 구현

Primitives 설계 철학

각 Primitive는 독립적으로 작동하면서도 조합 가능한 컴포저블 아키텍처를 따릅니다. 이는 UNIX 철학의 "Do one thing well"을 AI 컨텍스트에 적용한 것입니다.

🔧 Tools (도구)

{ "name": "database_query", "description": "Execute SQL query", "inputSchema": { "type": "object", "properties": { "query": {"type": "string"}, "params": {"type": "array"} }, "required": ["query"] }, "handler": async (params) => { // 트랜잭션 관리 // 쿼리 최적화 // 결과 스트리밍 } }

구현 패턴:

  • Idempotent 설계로 재시도 안전성 보장
  • JSON Schema 검증으로 타입 안정성 확보
  • Timeout과 취소 토큰 지원
  • 부분 결과 스트리밍 옵션

📁 Resources (리소스)

{ "uri": "file:///workspace/docs/*.md", "mimeType": "text/markdown", "metadata": { "size": 1024000, "modified": "2024-11-25T10:00:00Z", "permissions": "r--" }, "capabilities": { "watch": true, "stream": true, "paginate": true } }

URI 스키마 지원:

  • file:// - 로컬 파일시스템
  • db:// - 데이터베이스 리소스
  • api:// - 외부 API 엔드포인트
  • memory:// - 인메모리 데이터

💬 Prompts (프롬프트)

{ "name": "/analyze_codebase", "description": "Analyze project structure", "arguments": [ { "name": "path", "description": "Project root path", "required": true }, { "name": "depth", "description": "Analysis depth", "default": 3 } ], "template": "Analyzing {{path}} with depth {{depth}}..." }

고급 기능:

  • 템플릿 변수 바인딩
  • 조건부 프롬프트 렌더링
  • 다국어 지원
  • 컨텍스트 인젝션

🌲 Roots (루트)

{ "roots": [ { "path": "/app/data", "permissions": "rw-", "sandbox": true }, { "path": "/app/config", "permissions": "r--", "readonly": true } ], "policy": "strict", "escape": false }

보안 경계 설정:

  • Chroot 스타일 격리
  • 권한 상속 제어
  • 심볼릭 링크 추적 정책
  • 경로 이스케이프 방지

🎨 Sampling (샘플링)

{ "method": "sampling/request", "params": { "messages": [ {"role": "user", "content": "..."}, {"role": "assistant", "content": "..."} ], "modelPreferences": { "temperature": 0.7, "maxTokens": 2000, "stopSequences": ["\\n\\n"] } } }

활용 시나리오:

  • 코드 자동 완성
  • 문서 템플릿 채우기
  • 형식 변환 (MD → HTML)
  • 컨텍스트 기반 제안

❓ Elicitations (유도)

{ "type": "form", "schema": { "type": "object", "properties": { "apiKey": { "type": "string", "format": "password", "description": "API Key" }, "endpoint": { "type": "string", "format": "uri", "pattern": "^https://" } } }, "ui": { "submitLabel": "Connect", "cancelable": true } }

보안 주의사항:

  • 민감 정보는 환경변수 사용
  • 입력 마스킹 처리
  • 세션 종료 시 자동 정리
  • 감사 로그 기록

🔄 Primitives 조합 패턴

// 복합 워크플로우 예시 async function complexWorkflow(client) { // 1. Root 설정으로 작업 공간 제한 await client.setRoots(['/workspace']); // 2. Resource로 데이터 탐색 const files = await client.listResources('file:///workspace/*.json'); // 3. Tool로 데이터 처리 const results = await client.executeTool('process_json', {files}); // 4. Sampling으로 결과 요약 const summary = await client.requestSampling({ prompt: `Summarize: ${results}`, maxTokens: 500 }); return summary; }

🔄 MCP 수명주기 관리

1. 초기화 단계 (Initialization)

// 서버 초기화 시퀀스 { "phase": "initialize", "steps": [ "프로세스 생성/연결", "버전 협상 (0.1.0)", "Capabilities 교환", "인증 핸드셰이크", "세션 상태 초기화" ], "timeout": 30000, "retries": 3 }

2. 협상 단계 (Negotiation)

// 기능 협상 프로토콜 { "client_capabilities": { "tools": true, "resources": true, "experimental": ["streaming"] }, "server_capabilities": { "tools": true, "resources": true, "maxConcurrency": 10 } }

3. 운영 단계 (Operation)

  • 요청 처리: 병렬/순차 실행 모드

  • 에러 복구: 자동 재시도 메커니즘
  • Health Check: 주기적 상태 확인

4. 종료 단계 (Shutdown)

  • Graceful Shutdown: 진행중 작업 완료
  • 리소스 정리: 메모리/파일 핸들 해제
  • 상태 저장: 세션 데이터 영속화
  • 연결 종료: 클린 디스커넥트

// Zero Trust 보안 모델 구현 class MCPSecurityManager { async authenticate(request) { // 1. 신원 확인 const identity = await this.verifyIdentity( request.token, request.clientCert ); // 2. 장치 신뢰도 평가 const deviceTrust = await this.assessDevice( request.deviceId, request.attestation ); // 3. 컨텍스트 기반 위험 평가 const riskScore = await this.calculateRisk({ location: request.ip, time: request.timestamp, behavior: request.pattern }); // 4. 적응형 인증 if (riskScore > 0.7) { return this.requireMFA(identity); } return this.grantAccess(identity, deviceTrust); } }

심층분석 1: 기술적 깊이

JSON RPC 2.0 기반 설계의 의미

MCP가 RESTful API나 GraphQL 대신 JSON RPC를 선택한 것은 전략적 결정입니다.

{ "jsonrpc": "2.0", "method": "tools/execute", "params": { "tool": "create_mockup", "arguments": {...} }, "id": 1 }

장점: 양방향 통신, 실시간 상태 동기화, 비동기 작업 처리가 자연스럽게 가능합니다. 버전 협상과 기능 탐지가 내장되어 하위 호환성을 보장하면서도 점진적 발전이 가능합니다.

3계층 구조의 설계 철학

전송 계층의 유연성(stdio/HTTP), 데이터 계층의 표준화(JSON RPC), 애플리케이션 계층의 추상화가 완벽한 균형을 이루어 다양한 배포 시나리오에 대응합니다.

심층분석 8: 잠재적 함정

🚫 과도한 추상화

모든 시스템을 MCP로 통합하려는 시도는 오히려 복잡성을 증가시킵니다. AI 상호작용이 필요 없는 영역까지 MCP로 래핑하는 것은 불필요한 오버헤드입니다.

⚠️ 보안 위험

로컬 MCP 서버는 사용자의 전체 시스템 권한으로 실행됩니다. 악의적이거나 취약한 서버는 시스템 전체를 위험에 빠뜨릴 수 있습니다.

데이터 거버넌스 고려사항

  • LLM의 학습 데이터나 캐시를 통한 민감 정보 유출 위험
  • GDPR, CCPA 등 개인정보 보호 규정 준수 필요
  • 금융, 의료 등 규제 산업의 컴플라이언스 요구사항
  • 데이터 분류 및 접근 권한 관리 체계 수립
